# Plan: Scheduled Envoy Build via Tailscale SSH

Build Envoy on the Mac mini from a GitHub Actions scheduled workflow in
`dio/envoy-builder`. The runner (GitHub-hosted) joins the tailnet via the
`tailscale/github-action`, then SSHes to the mini using the existing
`envoy-mini-builder build --host` mechanism. No `--local` flag, no
self-hosted runner, no code changes to envoy-mini-builder.

## Context

- Mac mini: enrolled in tailnet, reachable as `mini` via MagicDNS
- envoy-mini-builder already handles the full SSH → build → scp → release flow
- Tailscale installed on the mini (confirmed)

See docs/tailscale-ci-setup.md for the full Tailscale + secrets setup guide.

## What needs to happen

### Tailscale admin (one-time)

1. Enable MagicDNS (https://login.tailscale.com/admin/dns)
2. Add `tag:ci` to ACL policy with an accept rule for `tag:ci` → `mini:22`
3. Create an OAuth client (scope: Auth Keys Write, tag: tag:ci)
   → yields TS_OAUTH_CLIENT_ID + TS_OAUTH_SECRET

### GitHub (one-time)

Add repository secrets in dio/envoy-builder:
  TS_OAUTH_CLIENT_ID, TS_OAUTH_SECRET, MINI_SSH_KEY, BUILDBUDDY_API_KEY

### New file: .github/workflows/scheduled-build.yml in dio/envoy-builder

```yaml
name: Scheduled Envoy Build
on:
  schedule:
    - cron: '0 2 * * *'
  workflow_dispatch:
    inputs:
      sha:
        description: 'Commit SHA, branch, or tag'
        default: 'main'
      platform:
        description: 'Platform (leave empty for all)'
        default: ''

jobs:
  build:
    runs-on: ubuntu-latest
    timeout-minutes: 180
    steps:
      - name: Connect to Tailscale
        uses: tailscale/github-action@v3
        with:
          oauth-client-id: ${{ secrets.TS_OAUTH_CLIENT_ID }}
          oauth-secret: ${{ secrets.TS_OAUTH_SECRET }}
          tags: tag:ci

      - name: Set up SSH key
        run: |
          mkdir -p ~/.ssh
          printf '%s\n' "${{ secrets.MINI_SSH_KEY }}" > ~/.ssh/id_ed25519
          chmod 600 ~/.ssh/id_ed25519

      - name: Install envoy-mini-builder
        run: |
          gh release download --repo dio/envoy-mini-builder \
            --pattern 'envoy-mini-builder_linux_amd64.tar.gz' \
            --output /tmp/emb.tar.gz
          tar -xzf /tmp/emb.tar.gz -C /usr/local/bin envoy-mini-builder
          chmod +x /usr/local/bin/envoy-mini-builder
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Build
        run: |
          SHA="${{ github.event.inputs.sha || 'main' }}"
          PLATFORM_FLAG=""
          if [[ -n "${{ github.event.inputs.platform }}" ]]; then
            PLATFORM_FLAG="--platform ${{ github.event.inputs.platform }}"
          else
            PLATFORM_FLAG="--all-platforms"
          fi
          envoy-mini-builder build \
            --sha "$SHA" \
            --host dio@mini \
            $PLATFORM_FLAG \
            --gh-repo dio/envoy-builder
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          BUILDBUDDY_API_KEY: ${{ secrets.BUILDBUDDY_API_KEY }}
```

Notes on install step: goreleaser produces `envoy-mini-builder_linux_amd64.tar.gz`
(check .goreleaser.yml for exact asset names — see below).

## Asset name check

From .goreleaser.yml the archive name pattern is:
  envoy-mini-builder_{{ .Os }}_{{ .Arch }}  (or similar)

Adjust the `--pattern` in the install step to match the actual release asset
name for linux/amd64. Alternatively: `go install` if the runner has Go:
  go install github.com/dio/envoy-mini-builder@latest

Or: add a linux/amd64 binary to the goreleaser config if not already present
(currently the dist/ shows darwin-arm64, linux-arm64, linux-amd64 — all
needed for building, the CLI itself needs linux-amd64 for the runner).

## Validation

1. Manually confirm SSH over Tailscale works from a non-LAN machine:
     ssh -i ~/.ssh/envoy_builder_ci dio@mini uname -m
2. Trigger the workflow via workflow_dispatch with sha=main, platform=darwin-arm64
   (shorter than --all-platforms for the initial test).
3. Check the GH Actions log for the Tailscale connect step — should show
   "Connected" and print the ephemeral node IP.
4. Confirm the build completes and the release asset appears in dio/envoy-builder.
5. Enable the cron schedule after the manual trigger succeeds.

## Risks / notes

- Build duration: 30-90 min per platform. --all-platforms is sequential in
  the tool; total wall time can exceed 3h. Set timeout-minutes: 240 or split
  platforms into matrix jobs.
- The mini must stay awake during the build. Prevent sleep:
    sudo pmset -a sleep 0 disksleep 0
  Or use an amphetamine-style keepawake LaunchAgent.
- Tailscale ephemeral nodes are automatically removed when the job ends.
  No manual cleanup needed.
- goreleaser: verify that linux-amd64 is in the build matrix so the runner
  can install the CLI itself. Current .goreleaser.yml should cover it.

## Step-by-step

1. Complete Tailscale + secrets setup per docs/tailscale-ci-setup.md.
2. Verify SSH over Tailscale from outside LAN.
3. Check .goreleaser.yml asset names → confirm linux-amd64 binary exists.
4. Create .github/workflows/scheduled-build.yml in dio/envoy-builder.
5. workflow_dispatch → single platform test.
6. On success: enable cron.
